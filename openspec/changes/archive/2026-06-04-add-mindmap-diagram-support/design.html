範圍
Goals
AI 能在 design.html 中正確選用 mindmap 呈現多維概覽
提供明確的 mindmap 語法規則，避免生成不合法 Mermaid
openspec-propose-design-mermaid-html 與 openspec-design-html 兩個 skill 規則與模板保持一致
Non-Goals
不升級 Mermaid CDN 版本（Mermaid 11 已原生支援）
不修改 page-shell.html 的 CSS / JS
不影響其他五個 skill
不合併兩個 page-shell.html 為共用檔案

技術決策 ／ 風險
技術決策
D1 ｜純加法，不重寫現有規則
在兩張表各補一列、Mermaid Rules 加獨立區塊，不動現有段落，零破壞性風險。
✗ 棄用：合併入 block-beta 列說明 → 語意模糊，AI 難以區分選用場景
D2 ｜Section Guide + 圖表表格雙觸發
兩張表同時新增 mindmap。Guide 決定「加不加 section」，表格決定「用哪種圖表語法」，缺一則 AI 不會主動選用。
D3 ｜mindmap 定位為 Scope 旁概覽，禁入 Before/After
mindmap 不支援 style，無法產生紅/綠對比節點；強放入 cmp-panel 會破壞視覺結構。
風險
⚠ label 過長導致 radial layout 破版
mindmap 節點文字超長時 SVG 渲染溢出或重疊。
→ SKILL.md 明確限制 label < 20 chars
⚠ AI 濫用 mindmap 取代 flowchart
mindmap 無方向性，誤用在有流程的場景會失去因果表達。
→ 觸發訊號限定「多維概覽」，與 flowchart「資料流 / 程式路徑」明確區隔
⚠ 兩個 page-shell.html 未來漂移
手動維護兩份相同檔案，後續修改易產生差異。
→ 此次同步更新（MD5 驗證一致）；共用單一 shell 為未來獨立 change
